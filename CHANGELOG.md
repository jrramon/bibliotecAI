# Changelog

## Unreleased

- Slice 9: Dashboard + library header polish — `libraries#index` becomes a card grid (mini-spine strip + title + excerpt + libros/miembros/fotos stats); `libraries#show` gets an eyebrow, serif page title, and a three-cell stat row above the book grid. Shared `hero-empty` empty state with kanji watermark replaces the old flat message.
- fix: deduplicate identified books per library on re-upload (Book.normalize).
- ops: drop Solid Queue for the identification path. The macOS arm64 pg-gem segfaults in forked Solid Queue workers even with `OBJC_DISABLE_INITIALIZE_FORK_SAFETY`, so `ShelfPhotosController#create` no longer enqueues through ActiveJob. Instead `bin/shelf-photo-poller` runs on the host, polls `ShelfPhoto.pending`, and invokes `BookIdentificationJob` inline (no fork). `bin/queue-status` reports photo state and whether the poller is alive. Storage reverts to a plain bind-mount so the host poller can read the blobs. ActiveJob adapter back to `:async` for the small in-process jobs (ActiveStorage analyze).
- Slice 8: Annotated shelf images — ShelfImageAnnotator service uses MiniMagick to draw red rectangles over the bounding boxes Claude flagged as unreadable, then attaches the result as `annotated_image` on the ShelfPhoto. Integrated into BookIdentificationJob so a completed run leaves both the identified Books and (when there are any unreadable spines) an annotated companion image. Shelf photo show page already renders it when attached. 2 system tests (with/without unidentified boxes). Session reset between system tests so Selenium cookies don't leak across runs.
- Slice 7: Claude identification job — ClaudeBookIdentifier service shells out to `claude -p` with `--output-format json` and the image path; the prompt asks for titles, authors, confidence, and bounding-box coordinates of unreadable spines. BookIdentificationJob is now fully wired: it updates the ShelfPhoto status through pending → processing → completed/failed, creates Book records for entries with confidence ≥ 0.5, stores the full response in `claude_raw_response`, and broadcasts the status badge over Turbo Streams. Image is written to `tmp/shelf_photos/` (not `Dir.mktmpdir`) so the CLI can read it through its working-directory sandbox; `--add-dir` and `chdir: Rails.root` are passed to Open3. Dev adapter switched to solid_queue; a migration loads the Solid Queue schema into the main database. README documents the host worker invocation with the macOS fork-safety env var.
- Slice 6: ShelfPhoto upload — ShelfPhoto model with status enum (pending/processing/completed/failed), ActiveStorage image + annotated_image attachments, and a jsonb claude_raw_response column. Upload form on the library page, status-badge page on show (subscribed to a Turbo Stream for future live updates). A stub BookIdentificationJob is enqueued on create; Slice 7 wires the actual Claude call. ActiveStorage uses MiniMagick (libvips isn't installed in the container). 1 system test.
- Slice 5: Comments with ActionText + Turbo Streams — Comment model with ActionText rich body, belongs_to :book and :user. Renders on book show with a compose form; new comments broadcast via `turbo_stream_from [@book, :comments]` so every viewer sees them appended live. Authors can delete their own comments (button guarded by a `viewer` local so broadcast partials don't leak the action to non-authors). 2 system tests.
- Slice 4: Books CRUD — Book model with library-scoped FriendlyId slugs (+history) and ActiveStorage `cover_image` (card + hero variants). Nested under libraries: index (grid w/ kanji-stamp placeholder covers), show (skewed hero cover + bibliographic table + edit/delete), new, edit. Content-type whitelist on uploads. 1 system test covers add → edit → delete.
- Slice 3: Email invitations — Invitation model with has_secure_token + 14-day TTL. Owners invite by email on the library page; the invitee receives a magic-link email via InvitationsMailer. Visiting `/invitations/:token` while signed-out stores the location, redirects to sign-up with the email prefilled (Users::RegistrationsController override), and Devise restores the link after sign-up so acceptance happens in one click. Spanish locale added. 2 new system tests cover the full two-user flow end-to-end.
- Design: Adopt the Hakobune design system (warm wood & washi, Japanese bookshop aesthetic). Adds CSS variable tokens (washi/sepia/sumi-dark themes), Fraunces/Inter/JetBrains-Mono typography, paper-texture overlay, 函 brand mark, sidebar + header app shell, styled hero, dashboard cards, auth forms and library pages.
- Slice 2: Libraries + memberships — Library (FriendlyId slug w/ history) and Membership (role enum: owner/member). Owner membership auto-created on Library create. Dashboard (`/libraries`) as authenticated root. Scoped access; no cross-tenant leaks. 2 system tests.
- Slice 1: Devise authentication — User model, sign up / sign in / sign out, Spanish nav, letter_opener_web for dev mail, 2 system tests.
- Slice 0: Scaffold & infrastructure — Rails 8 app, Docker Compose dev, CI, linters, Devise/Solid trio/ActionText/ActiveStorage wired.
